集合类别

Collection

Collection 是最基本的集合接口，一个 Collection 代表一组 Object，即 Collection 的元素, Java不提供直接继承自Collection的类，只提供继承于的子接口(如List和set)。Collection 接口存储一组不唯一，无序的对象。Collection集合方法

常用集合分类

Collection 接口的接口 对象的集合（单列集合）
├——- List 接口：元素按进入先后有序保存，可重复，通过索引操作元素
│—————-├ LinkedList 接口实现类， 双向链表， 没有同步，查询慢，增删快，线程不安全，效率高，可以存储重复元素
│—————-├ ArrayList 接口实现类， 数组， 随机访问，查询快，增删慢，效率高，可以存储重复元素， 没有同步， 线程不安全
│—————-└ Vector 接口实现类 数组， 同步，查询快，增删慢，线程安全，效率低，可以存储重复元素
│ ———————-└ Stack 是Vector类的实现类
└——- Set 接口：hash表，不可重复，并做内部排序，不能根据索引获取元素，线程不安全，效率高，可以存储null元素， 元素的唯一性是靠所存储元素类型是否重写hashCode()和equals()方法来保证的，如果没有重写这两个方法，则无法保证元素的唯一性。
├—————-└ HashSet 使用hash表（数组）存储元素
│————————└ LinkedHashSet 构采用链表和哈希表共同实现，链表保证了元素的顺序与存储顺序一致，哈希表保证了元素的唯一性。线程不安全，效率高。
└ —————- TreeSet 底层实现为二叉树，元素排好序，需要重写hashCode和equals()方法

HashMap实现原理

HashMap类中的元素就是节点
HashMap通过key的hashCode来计算hash值，不同的hash值就存在数组中不同的位置，当多个元素的hash值相同时

Node类的基本属性有：

  • hash：key的哈希值
  • key：节点的key，类型和定义HashMap时的key相同
  • value：节点的value，类型和定义HashMap时的value相同
  • next：该节点的下一节点
    重写equals()和hashCode()方法

散列表

链表和数组都是按顺序存储的，相当于是有序的。这样就会导致一个缺点，访问某个元素就要访问所有的元素，会加大时间复杂度。散列表就可以改变这种情况。

hash表原理

散列表为每个对象计算出一个整数，称为散列码。根据这些计算出来的整数(散列码)保存在对应的位置上。
可能会遇到散列码相同，这种情况叫做散列冲突，是不可避免的。
如果散列表太满，需要再散列，创建新的散列表，将原有的元素插入新表中。
转载因子（load factor）（默认值0.75）决定 什么时候创建新的表。如果表中元素超过了75%，则按照两倍的表再散列。

红黑树

JDK1.8 散列表的链表太满就会转换成红黑树存储。TreeSet和TreeMap底层都是红黑树。
红黑树是基于 2-3树 基于平衡树的结构。2-3树结构

2-3树保持树的平衡十分复杂，需要大量的交换节点。

红黑树结构


红黑树取代2-3树交换节点的方法:

  • 旋转：顺时针和逆时针
  • 反色：交换红黑的颜色

另外，红黑树还有一些约束：

  • 红黑树是二叉搜索树。
  • 根节点是黑色。
  • 每个叶子节点都是黑色的空节点（NIL节点）。
  • 每个红色节点的两个子节点都是黑色。(从每个叶子到根的所有路径上不能有两个连续的红色节点)
  • 从任一节点到其每个叶子的所有路径都包含相同数目的黑色节点(每一条树链上的黑色节点数量必须相等)。
