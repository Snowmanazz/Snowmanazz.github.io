redis概述

redis消息队列–stream

​ Stream拥有很高的性能和内存利用效率，提供了消息的持久化存储，以及主从复制功能，从而解决网络断开、redis宕机、消息丢失等问题。

​ Stream消息队列由消息本身、消费者、生产者和消费组组成。

  • **Stream Direction:**表示数据流，是一个消息链，将所有消息串联起来

  • **Consumer Group:**消费组，一条StreamDirection上有多个ConsumerGroup，一个ConsumerGroup包含多个Consumer。
  • **last_delivered_id:**消费者游标，每个ConsumerGroup都有一个last_delivered_id，任意消费者读取都将会使游标前移。
  • **pending_ids:**消费者的状态变量，记录当前已被读取但没有被ACK的消息。

消息ID唯一，系统创建则为*

redis事务

redis事务特性：

  • 单独隔离操作：事务中的命令会被序列化，按照顺序执行，不会被其他命令打断执行。
  • 不保证原子性：如果命令执行失败，其他命令依旧执行，不支持事务回滚。

RedisTemplate操作redis

Spring 封装了 RedisTemplate 来操作 Redis，它支持所有的 Redis 原生的 API。在 RedisTemplate 中定义了对5种数据结构的操作方法。

  • opsForValue()：操作字符串。
  • opsForList()：操作列表。
  • opsForHash()：操作哈希。
  • opsForSet()：操作集合。
  • opsForZSet()：操作有序集合。

RedisConfig类：

@Configuration
public class RedisConfig {
    /**
     * redisTemplate 序列化使用的jdkSerializeable, 存储二进制字节码, 所以自定义序列化类
     * @param redisConnectionFactory redis连接工厂类
     * @return RedisTemplate
     */
    @Bean
    public RedisTemplate<Object, Object> redisTemplate(RedisConnectionFactory redisConnectionFactory) {
        RedisTemplate<Object, Object> redisTemplate = new RedisTemplate<>();
        redisTemplate.setConnectionFactory(redisConnectionFactory);

        // 使用Jackson2JsonRedisSerialize 替换默认序列化
        Jackson2JsonRedisSerializer jackson2JsonRedisSerializer = new Jackson2JsonRedisSerializer(Object.class);

        ObjectMapper objectMapper = new ObjectMapper();
        objectMapper.setVisibility(PropertyAccessor.ALL, JsonAutoDetect.Visibility.ANY);
        objectMapper.enableDefaultTyping(ObjectMapper.DefaultTyping.NON_FINAL);

        jackson2JsonRedisSerializer.setObjectMapper(objectMapper);

        // 设置value的序列 化规则和 key的序列化规则
        redisTemplate.setKeySerializer(new StringRedisSerializer());
        redisTemplate.setValueSerializer(jackson2JsonRedisSerializer);
        redisTemplate.afterPropertiesSet();
        return redisTemplate;
    }
}
